Code used in the "Common Control"

Below are commonly used code snippets for use in the Common Control section of MOM.

Conditional Assignment

$x = 4 if ($a == 2);

This assigns the value 4 to $x only if the condition $a == 2 is true. In the Common Control environment, this shorthand syntax works for simple one-line conditionals.

For more complex logic, you can use a full if...else structure like this:

if ($a > 10) {
  $x = 100;
} else {
  $x = 0;
}

This checks whether $a is greater than 10 and assigns a value to $x accordingly.

Declare answer types for multipart

$anstypes = array("number", "choices");

You can declare different input types for each part of a multipart problem using strings like "number", "choices", "multans" (multiple answer checkboxes), "calculated" (MathQuill input), "string", or "essay" Each entry in the array corresponds to one part of the question.

Assign weights to multipart parts

$answeights = array(0.5, 0.5);

This controls how much each part of a multipart question contributes to the total score. For example, $answeights = array(0.5, 0.5); means the first part will be worth 50% of the points and the second 50%. You can also assign different weights, like array(0.4, 0.4, 0.2) â€” here, the first two parts are each worth 40% and the third is worth 20%.

Declare answer value(s)

$answer[0] = "5";

If the problem has only one part or answer box, you can simply use $answer = "5";. For multipart problems, use indexed answers like $answer[0] for the first part, $answer[1] for the second part, and so on. Each value should match the corresponding $anstypes definition.

Declare choices for a question

$choices[1] = array("Yes", "No");

The $choices array defines the answer options for a multiple-choice question. In a multipart problem, each index corresponds to one part: for example, $choices[0] provides the choices for $answerbox[0], while $choices[2] defines the choices for the third part of the problem ($answerbox[2]). Be sure each $choices[x] array matches the corresponding $anstypes[x] as "choices".

Control formatting of answer display

$displayformat = "decimal";

The $displayformat setting controls how the student's answer is shown in the answer box. Common options include:

  • "decimal" â€“ Displays the input as a plain decimal number (default for numerical questions).
  • "mixednumber" â€“ Allows or requires a mixed number format like 1 1/2.
  • "fraction" â€“ Displays a proper or improper fraction like 3/4 or 7/5.
  • "scinot" â€“ Displays numbers in scientific notation (e.g., 3.2e5).
  • "sigfig" â€“ Enforces a specific number of significant figures (requires $sigfigs to be set).
  • "logic" â€“ For logical expressions (true/false logic).
  • "list" â€“ For list-style answers (e.g., entering multiple numbers separated by commas).
  • "string" â€“ For free-response string input (used with $anstypes = "string").
  • "typein" â€“ A basic text box (used with "string" type).
  • "noinput" â€“ Hides the answer box entirely (used for display-only questions).

Choose the format that best matches your question's expectations and grading method.

Require decimal precision

$reqdecimals = 2;

This setting enforces that a studentâ€™s answer is entered with a specific number of decimal places. For example, $reqdecimals = 2; means all answers must have exactly 2 decimal places. To require precision on just one part of a multipart question, use an index like $reqdecimals[1] = 3; to enforce 3 decimals on only the second part.

Set absolute tolerance

$abstolerance = 0.01;

This sets how far a student's numerical answer can deviate from the correct answer and still be marked correct. For example, $abstolerance = 0.01; allows answers within Â±0.01 of the correct value. For multipart questions, you can apply it to just one part using an index like $abstolerance[0] = 0.05;.

Set relative tolerance

$reltolerance = 0.001;

This sets the tolerance as a percentage of the correct answer, rather than a fixed amount. For example, $reltolerance = 0.001; allows answers within 0.1% of the correct value. To apply this to only one part of a multipart question, use an index like $reltolerance[2] = 0.01; to allow a 1% margin on the third part.

Load custom macro library

loadlibrary("stats");

The loadlibrary() function allows you to bring in additional macros and functions that expand what you can do in the Common Control section. You can load multiple libraries at once by separating them with commas, like loadlibrary("stats,fractions");.

For example, the "stats" library gives you access to functions such as:

  • mean(array) â€“ Computes the mean of a data set
  • stdev(array) â€“ Computes the standard deviation
  • percentile(array, p) â€“ Finds a percentile
  • histogram(array,...) â€“ Generates histogram display data
  • linreg(xarray, yarray) â€“ Performs linear regression

Another useful library is "fractions", which enables fraction-specific operations and representations (like handling reduced forms). Load only the libraries you need to keep your problem lightweight and avoid unnecessary function conflicts.

Generate random int from range

$x = rand(1, 10);

This generates a random integer value between 1 and 10, inclusive: $x = rand(1, 10);

To generate a random number between 1 and 10 to one decimal place, you can do: $x = rand(10, 100) / 10;
This generates a random integer from 10 to 100 and then divides by 10 to produce values like 1.2, 7.5, or 9.8.

Generate random float from range

$r = rand(100, 200)/100;

This technique is useful when you want to generate decimal values (floats) with consistent formatting for use in real-world contexts like prices, measurements, probabilities, or statistical data. For example, $r = rand(100, 200)/100; produces values between 1.00 and 2.00, which are commonly needed in statistics, economics, and applied math problems.

Choose random decimal from array

$alpha = $alphas[rand(0,2)]; // $alphas = array(0.01, 0.05, 0.10);

This code selects a random value from the $alphas array. The expression rand(0,2) generates a random index (0, 1, or 2), and $alphas[...] retrieves the corresponding value â€” in this case, one of 0.01, 0.05, or 0.10.

To randomly select multiple values from an array, you can shuffle the array and then pick the first few elements. For example: $alphas = array(0.01, 0.05, 0.10, 0.15);
shuffle($alphas);
$alpha1 = $alphas[0];
$alpha2 = $alphas[1]; This randomly selects two distinct alpha values from the original list.

Create array of random values

$a,$b = diffrands(1,10,2);

This code uses diffrands(1,10,2) to generate two different random integers between 1 and 10. The values are stored in $a and $b. This is useful when you need multiple values that are guaranteed to be distinct.

Round variable $r to decimal places

$z = round($r, 2);

This code rounds the value of $r to 2 decimal places and stores the result in $z. You can also use rounding directly in the answer declaration, such as $answer = round($r, 2); for a single-part problem, or $answer[0] = round($r, 3); to round to 3 decimal places in a multipart problem.
